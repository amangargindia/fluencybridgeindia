How The System Works

The Fluency Bridge Transcription Tool operates entirely client-side within your browser. By leveraging WebAssembly (WASM) and Transformers.js, the tool downloads the Whisper AI model directly to your device and processes audio locally.

Zero server dependency means your audio data never leaves your computer, guaranteeing 100% privacy and averting costly API usage.

Core Features

  • Live Dictation Typewriter: Streams live speech using the Web Speech API while recording, later compiling a highly accurate offline transcription using Whisper as a fallback.
  • Automated Extraction: The system automatically extracts plaintext and downloads a .txt file naming it by the current date and time upon transcription completion.
  • Instant Clipboard Sync: Transcriptions are automatically copied to your clipboard to negate accidental data loss.
  • Audio Export: Captured live audio can be downloaded from the transcript cards directly in .webm format.

Technicalities & Models

The system utilizes three Whisper architectural layers:

LayerSize (RAM needed)PerformanceUse Case
Tiny Layer~40MB Model (<1GB RAM)FastestLow-end devices, real-time fallback
Base Layer~145MB Model (~1.5GB RAM)BalancedRecommended. Optimal for general use.
Small Layer~500MB Model (>3GB RAM)HeavyHigh-end Desktop PCs, complex vocabularies.

Caveats & Limitations

  • Mobile Device Constraints: Because WASM executes in browser memory, older iPhones or Androids may experience browser crashes if the Small model is used. Default to Base or Tiny.
  • Background Tab Throttling: Most browsers heavily throttle Javascript in inactive tabs. If you switch tabs during extraction, it will become dramatically slower or pause entirely until you return. Leave the tab active while engaged!
  • Initial Download Lag: The very first time you invoke the engine, it must download the ML weights into your cache (up to 145MB). Future visits will utilize the cached weights instantly.
